Explain dependency injection in Swift with an example
Dependency injection is a pattern where an object receives its dependencies from outside rather than creating them internally. This makes code testable and loosely coupled.

Initializer injection (the most common form in Swift):
protocol API { func fetch() }

class Service {
  let api: API
  init(api: API) { self.api = api }
}

// Test with mock
class MockAPI: API { ... }
Now Service works with any API implementation — real or mock.

Frequently asked.

Common questions about HideAIChat.

Do I need a separate API key?
No. HideAIChat uses your Notion AI subscription through notion2api. The only credential you need is your Notion token_v2 cookie.
Is it free?
The app and notion2api are both open source and free. You only pay for your Notion AI subscription (already required).
Is it safe?
All communication stays on your machine. The notion2api server runs locally on port 8000 and never sends your token elsewhere.
Does it work with Apple Silicon?
Yes. Optimized for Apple Silicon (arm64). macOS 15+ required due to SwiftUI and Swift 6 APIs used.
Can I use other AI models?
HideAIChat exposes all models available through notion2api — Claude Sonnet/Opus, GPT-5.x, Gemini 2.5/3.1, and Kimi 2.6.
What about privacy?
Your conversations are stored locally in chat_history.json. No data is sent to any third party besides Notion through notion2api.
